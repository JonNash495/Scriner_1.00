2) Из получаемых данных (например для формата json) нужно отобрать следующие ключи их значения:
- символ (например BTCUSDT)
- последняя цена 
- цена продажи 
- цена покупки 
- комиссия за покупку
- комиссия за перевод на другую биржу
3) Отобранные данные нужно представить в двух вариантах:
1. табличный формат данных, где маркеры столбцов - биржи, а маркеры строк - валютные пары. На пересечении находится информация о парах из п.2

"данные": {
	"валютная пара_1": {"биржа_1": {информация о "валютная пара_1" из п.2},

				...

				...
			   },

}

Преобразования ответов:
- Никакие символы не изменять и не добавлять
- Если ключ-валюта пришел в нижнем регистре, привести к верхнему
- Если между валютами в валютной паре есть разделители - устранить
- пример: (10000LADY_USDT -> 1000LADYUSDT, 1000lady_usdt -> 1000LADYUSDT)
- Преобразования ключей во вложенном словаре валюты:
Все ключи получаемые от биржи нужно приводить к следующему шаблону: 
(пример) 
"binance":  { "lastPrice": 0.001, "bidPrice": 0.002, "askPrice": 0.00099} 
То есть, если, например, ключ для последней цены предствлен как "last_price", то необходимо преобразовать его к виду "lastPrice". Аналогично для "bidPrice" и "askPrice"



Информация о ключах:   
Binance: askPrice = цена покупки, bidPrice = цена продажи, lastPrice = последняя цена, symbol или marker = название котировки. 
На разных биржах ключи могут отличаться, но они созвучны с этим шаблоном.
